# The Orchestrator — Claude Code Project Instructions

An autonomous PM-to-PL orchestration framework for Claude Code. Plans sprints from outcomes, executes them with quality gates, and ships code.

## Quick Start

See `docs/QUICK_START.md` for setup instructions.

## Architecture

See `docs/ARCHITECTURE.md` for the AI-Human Engineering Stack and how The Orchestrator uses each layer.

## Skills (Slash Commands)

Skills live in `.claude/skills/`. Each skill is a markdown file that defines a Claude Code slash command.

| Skill | Purpose |
|-------|---------|
| `/orchestrate` | Launch the autonomous PM-to-PL cycle |
| `/outcomes` | Define project outcomes interactively |
| `/prd` | Generate a Product Requirements Document |
| `/architect` | Design architecture from a PRD |
| `/taskgen` | Generate structured task list from PRD |
| `/execute` | Orchestrate wave-based task execution |
| `/reliability-audit` | Identify failures, gaps, and missing tests |
| `/code-quality-review` | Review code changes for quality |
| `/commit` | Stage and commit with conventional message |
| `/ship` | Full workflow: branch, commit, push, PR |
| `/validate` | Comprehensive validation pipeline |
| `/values-discovery` | Discover core values and engineering principles |
| `/drift-report` | Weekly code quality pattern analysis |
| `/update` | Auto-update memory files from git diffs |
| `/delta` | Minimum variance analysis |
| `/investigate` | Pre-implementation feasibility research |
| `/feature` | AI-optimized feature implementation |
| `/bugs` | Bug exploration and fix planning |

## Directory Structure

```
.claude/
  skills/       # Slash command definitions (one SKILL.md per command)
  rules/        # Path-triggered convention rules (auto-loaded)
  VALUES.md     # Your project's identity and decision heuristics (generated by /values-discovery)
  SKILLS_GUIDE.md  # Decision tree for choosing the right skill

.ai/
  CONTEXT.md    # Project overview (your project's context)
  retros/       # Sprint retrospectives (generated per sprint)
  audits/       # Reliability and security audit reports
  patterns.md   # Cross-run learnings (auto-maintained)
  investigations/  # Feasibility research reports

shared/
  OUTCOMES.md   # Project outcomes (the "what to build")
  ROADMAP.md    # Sprint statuses and progress
```

## Rules

Rules in `.claude/rules/` auto-load based on file path triggers. They enforce conventions without requiring manual invocation.

| Rule | Purpose |
|------|---------|
| `values-discovery.md` | Reuse before create — discovery process for existing patterns |
| `test-requirements.md` | No shipping without tests — test gates for pure functions |
| `event-transparency.md` | Every async operation needs a visible UI signal |

## State Files

The Orchestrator reads and writes these state files:

- **`shared/OUTCOMES.md`** — Project outcomes with measurable criteria. The PM agent reads this to plan sprints.
- **`shared/ROADMAP.md`** — Sprint statuses. Updated as sprints complete.
- **`.claude/VALUES.md`** — Your identity, decision heuristics, and code principles. Generated by `/values-discovery`. All agents reference this for personalized decision-making.
- **`.ai/patterns.md`** — Cross-run learnings. Auto-maintained after every 3rd sprint retro.

## Getting Started

1. Run `/values-discovery` to generate your `.claude/VALUES.md`
2. Run `/outcomes` to define what you want to build
3. Run `/orchestrate` to start the autonomous PM-to-PL cycle

The orchestrator will plan sprints, make architecture decisions, execute with quality gates, and ship code — all autonomously.
